for range 临时变量地址变化

在for i, v := range a中，变量i和变量v的内存地址是不会变的，每次循环会把新的值赋给i和v。因此，在for循环中如果要开启协程的话不能直接使用i和v的地址。

GoLang中make和new的区别

共同点

  • 都是用来分配内存。

  • 分配的位置都取决于逃逸分析。逃逸分析的核心思想是：如果变量的生命周期超出函数作用域，它会逃逸到堆上，否则可以安全地分配在栈上。

不同点

  • 作用对象不同，make作用在slice，map，channel上，new作用在其他的类型上。

  • 返回对象不同，make返回类型本身，而new返回一个指向该类型对象的指针。

使用 go build -gcflags="-m" 命令查看逃逸分析信息。

new一个变量，变量分配在堆上，原因是这个指针超出了该变量的作用域（被fmt.Println()调用）。

new一个变量，变量分配在栈上，原因是这个指针只在该函数的作用域使用。

make分配在堆上还是栈上跟new原理类似。

GoLang并发编程哲学之一：不要通过共享内存来通信，而要通过通信来共享内存

这是 Go 并发编程最核心的哲学，也是 Go 语言的 CSP 模型（Communicating Sequential Processes） 的关键思想。

  • 传统模型：大多数语言的并发编程（如 Java、C++）是通过线程共享内存（变量）实现的，需要通过锁、互斥量（Mutex）来保证数据一致性。这容易导致竞争条件和死锁问题。
  • Go 的哲学：
    • 使用 Goroutine 和 Channel（通道）来实现通信。
    • Goroutine 是轻量级线程，而 Channel 是线程安全的通信机制。
    • 通过 Channel 传递数据，而不是多个 Goroutine 直接操作共享的内存。

避免通过共享内存来进行通信的原因主要在于以下几个方面：

1. 避免数据竞争（Race Condition）

  • 直接共享内存时，如果多个 Goroutine 同时读写共享数据，没有适当的同步，就会导致数据竞争，最终导致程序行为不一致或崩溃。
  • 使用 Channel 传递数据时，每次的数据传递都是原子性的，发送者和接收者之间的同步由 Go 的运行时和 Channel 本身保证，不需要开发者手动加锁。

2. 减少复杂性

  • 使用共享内存时，通常需要手动管理同步机制，如使用 sync.Mutex、sync.RWMutex 等锁来避免竞争条件，增加了程序的复杂性和出错的概率。
  • Go 的 Channel 自带同步功能，简化了并发编程的设计，让开发者专注于业务逻辑，而无需关注底层细节。

3. 提高可维护性和可读性

  • 使用 Channel 进行通信，使得 Goroutine 之间的协作更加清晰和直观。通过数据流动的形式，程序员可以轻松追踪和理解不同 Goroutine 之间的数据传递过程。
  • 相比直接共享内存和显式加锁，Channel 提供了一种更为结构化的方式来组织并发逻辑。

4. 避免死锁（Deadlock）和锁的过度使用

  • 如果直接共享内存，程序员通常需要小心避免死锁。Go 通过 Channel 提供的同步机制，消除了这种风险。
  • Channel 提供了一种自带同步的机制，使得 Goroutine 之间的同步变得显而易见和简单。

这种方式有一定的缺点：

  • 不灵活，使用锁的方式可以随时随地的加锁解锁，而这种方式不支持。

Go中如何实现继承

使用组合模式。

思考：为什么Go中不提供类似于extends的关键字呢？

  • 简化设计：

    • Go 设计上追求简洁和明确，避免了传统 OOP 语言中继承所带来的复杂性和潜在问题（如多重继承、虚方法表、父类修改影响子类等）。
    • 通过组合和接口的方式，Go 保持了面向对象的特性，同时去掉了繁琐的继承和类层次结构，使得代码结构更加清晰和简单。

  • 避免复杂的继承体系：

    • 继承通常会带来复杂的类层次结构和潜在的依赖关系，这些在大型系统中可能会导致代码难以理解、维护和扩展。
    • Go 鼓励通过组合来复用代码，减少了父子类之间的耦合关系。你可以把一个结构体嵌套到另一个结构体中，从而复用其字段和方法，而不需要使用 extends 创建继承关系。

  • 组合和接口可以提供更多灵活性：

    • 组合：通过将一个结构体嵌套到另一个结构体中，你可以复用父结构体的功能，而不需要继承。组合的好处是能够避免继承中的单继承限制，同时更容易实现松耦合设计。
    • 接口：Go 通过接口提供了多态性。类型只需要实现接口的方法，就可以满足多态要求，而无需显式地声明“继承”。这种方式比传统的继承更加灵活和强大。

  • 消除多重继承的复杂性：

    • 在传统的面向对象语言中，多重继承可能导致“菱形继承问题”（diamond problem），即多个父类拥有相同的属性或方法，子类不知道继承哪个父类的版本，可能导致二义性或冲突。
    • Go 避免了这个问题，因为它不允许多重继承，而是通过组合和接口实现了类似多重继承的效果。

  • 减少方法冲突：

    • 传统继承中，子类可能会无意中覆盖父类的方法，造成方法冲突。在 Go 中，方法是通过组合来实现的，不会直接覆盖父类的方法，因此减少了这类问题。
    • 通过嵌套和组合，可以明确指定结构体的方法，不会发生无意间覆盖的情况。
